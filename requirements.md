# Yami 创作者计划 — 需求文档

## 简介

Yami 创作者计划是亚米电商平台的双层创作者变现系统。创作者通过个性化追踪链接分享和推广亚米商品来赚取佣金。计划分为两个层级：**Affiliate**（开放注册，低佣金）和 **Influencer**（需申请审核，高佣金 + 额外权益）。系统涵盖用户端前端（落地页、注册流程、工作台）和内部管理后台（创作者管理、打款、样品、推荐商品、佣金管理等）。

## 术语表

| 术语 | 定义 |
|---|---|
| **创作者（Creator）** | Yami 创作者计划的注册用户，分为 Affiliate 或 Influencer |
| **Affiliate** | 开放注册层级，无粉丝要求 |
| **Influencer** | 需 5000+ 粉丝 + 管理员审核，享有更高佣金比例及额外权益 |
| **追踪链接（Tracking Link）** | 包含创作者唯一标识的 URL，用于归因点击和购买 |
| **集合页（Collection）** | 创作者策划的商品合集，可通过单一链接分享，最多 50 个商品 |
| **Power Creator 标签** | 紫色星星徽章，仅活跃 Influencer 可获得并展示 |
| **活跃状态（Active Status）** | 滚动 3 个月内确认销售额 ≥ $300 即为活跃 |
| **佣金比例（Commission Rate）** | 按品类配置的销售额分成比例。每位创作者有默认佣金比例，管理员可为单个创作者手动调整 |
| **打款（Payout）** | 佣金发放，需提交税表后方可处理 |
| **样品（Sample）** | 以 $0 成本提供给符合条件的 Influencer 的商品 |
| **分享弹窗（Share Sheet）** | 提供复制链接、社交分享、添加到集合页的 UI 弹层 |
| **国家语言切换器（Locale Switcher）** | 选择国家（美国/加拿大）和语言的右侧抽屉组件 |
| **管理后台（Admin Dashboard）** | Yami 运营人员使用的内部管理界面 |
| **落地页（Landing Page）** | 面向公众的营销页面，介绍 Affiliate 或 Influencer 计划 |
| **税表（Tax Form）** | 创作者在收到打款前必须提交的法律文件（如 W-9） |
| **领取时间（Claim Period）** | 样品商品可供 Influencer 领取的日期范围 |
| **置顶商品（Pinned Product）** | 管理员手动选择的推荐商品，带有效期 |
| **优先佣金（Priority Commission）** | 为特定三级分类或 SKU 设置的、在指定时间段内覆盖默认佣金比例的专项佣金规则 |
| **归因窗口（Attribution Window）** | 从点击追踪链接到成交的有效归因时间，首单 7 天、复购 3 天 |
| **Last Click 归因** | 用户点击多个 influencer 链接时，以最近一次点击的 influencer 进行归因的排重规则 |

## 一、Landing Page

### 需求 1：Affiliate 落地页

**用户故事：** 作为潜在 Affiliate，我希望看到一个专属落地页，了解 Affiliate 计划的权益和注册流程，以便做出加入决定。

---

### 需求 2：Influencer 落地页

**用户故事：** 作为潜在 Influencer，我希望看到一个突出高级权益和申请要求的落地页，以便了解价值主张并申请。

---

### 需求 3：国家语言切换器

**用户故事：** 作为创作者或潜在创作者，我希望选择国家和语言，以便用母语和正确货币浏览平台。

**验收标准：**
1. 支持两个国家：美国（USD）和加拿大（CAD）
2. 支持语言：English、简体中文、繁體中文、한국어、日本語
3. 选择加拿大时，额外提供 Français 选项
4. 选择国家后，货币自动设置为对应值，置灰不可选
5. 在所有页面的吸顶导航栏中可访问

---

### 需求 4：Affiliate 注册流程

**用户故事：** 作为新 Affiliate 申请者，我希望完成简单的注册流程，立即开始分享商品赚取佣金。

**验收标准：**
1. 第 1 步：选择至少一个社媒平台并输入主页 URL或用户名
2. 第 2 步（品类偏好）：选择一个或多个一级品类
3. 提交后立即获得工作台访问权限，无需管理员审核
4. Affiliate 不需要填写粉丝数量

**错误处理：**
- 未选平台 → 报错
- 无效 URL → 对应输入框显示验证错误
- 未选品类 → 显示验证错误「请至少选择一个品类」

---

### 需求 5：Influencer 注册流程

**用户故事：** 作为新 Influencer 申请者，我希望完成注册流程并提交社媒信息，以便 Yami 评估我的资格。

**验收标准：**
1. 第 1 步：选择至少一个社媒平台并输入主页 URL或用户名
2. 需要选择粉丝数量范围
3. 第 2 步（品类偏好）：选择一个或多个一级品类
4. 提交后进入「审核中」状态，等待管理员审批
5. 审核通过后获得 Influencer 工作台访问权限
6. 审核拒绝后显示「已拒绝」状态页，展示拒绝原因，并提供联系业务邮箱的功能
7. 拒绝状态页提供「已知晓」按钮，点击后标记为已读；之后再次访问时直接进入 Landing Page，不再重复展示拒绝页

**错误处理：**
- 未选平台 → 报错
- 无效 URL → 对应输入框显示验证错误
- 未选品类 → 显示验证错误「请至少选择一个品类」

---

## 二、工作台

### 需求 6：工作台首页

**用户故事：** 作为创作者，我希望在工作台看到业绩数据和推荐商品，以便监控收入并发现新的推广商品。

**验收标准：**
1. 展示数据卡片行：总收入、本月收入、总订单数、转化率（大屏 4 列，小屏 2 列）
2. 展示业绩趋势图，支持收入/订单等多维度切换和时间范围选择
3. 展示推荐商品模块
4. Influencer 活跃用户展示免费样品模块；Affiliate 不展示
5. Influencer 活跃用户在问候区展示 Power Creator 标签；Affiliate 不展示
6. **Commission 活动 Banner**：当存在生效中的优先佣金活动时，在工作台首页展示活动 Banner，让创作者可以看到当前有哪些品类/商品正在享受更高佣金比例及活动截止时间；点击 Banner 跳转至 Commission 活动明细页（见需求 7）

---

### 需求 7：Commission 活动明细页

**用户故事：** 作为创作者，我希望查看当前所有进行中的佣金活动详情，以便了解哪些商品佣金更高，并优先推广这些商品。

**验收标准：**

**页面整体：**
1. 页面标题区展示活动说明文案，引导创作者重点推广活动商品
2. 仅展示状态为「进行中」的优先佣金活动；已结束或未开始的活动不展示

**活动列表：**
3. 每条活动以卡片形式展示，包含：
   - 活动名称 / 品类或 SKU 名称
   - 优先佣金比例（突出显示，与默认比例形成对比，如「默认 5% → 活动 12%」）
   - 活动截止时间（含倒计时或截止日期）
   - 适用范围说明（三级分类 or 指定 SKU）
4. 支持按品类筛选活动列表
5. 活动按截止时间升序排列（即将结束的排在前面）

**商品入口：**
6. 每条活动卡片提供「Share」按钮
7. 若活动对应的是三级分类，跳转该分类结果页
8. 若活动对应的是指定 SKU，直接跳转至该商品的分享弹窗

---

### 需求 8：商品分享与追踪链接

**用户故事：** 作为创作者，我希望用个性化追踪链接分享商品，以便通过我的链接产生的购买归因到我名下计算佣金。

**验收标准：**
1. 分享商品时生成包含创作者标识的唯一追踪链接
2. 分享弹窗提供：复制链接、社交平台分享、添加到集合页（可添加到已创建的集合页，或创建新的集合页）
3. 分享弹窗可从工作台所有商品卡片触发
4. 通过追踪链接产生的购买归因到对应创作者

---

### 需求 9：集合页（Collections）

**用户故事：** 作为创作者，我希望将商品分组为可分享的集合页，以便策划主题商品列表并通过单一链接分享给粉丝。

**验收标准：**
1. Affiliate 和 Influencer 均可创建集合页
2. 创建时需填写标题，描述可选
3. 每个集合页最多可添加 50 个商品，可添加当前页推荐单品、通过"添加商品"入口搜索并添加全网商品。已添加的商品支持顺序调整、删除功能。
4. 为每个集合页生成带追踪链接的分享 URL
5. 集合页展示统计数据：总点击、总订单、总佣金等
6. 集合页必填项已完成后，显示预览按钮，点击可预览集合页
7. 分享集合页时提供复制链接和社交平台分享选项
8. 粉丝访问集合页链接时展示分享人、标题、描述和商品卡片

**错误处理：**
- 超过 50 个商品 → 提示「已达商品数量上限」
- 无标题 → 显示验证错误「标题为必填项」
- 无效集合页链接 → 展示友好的「集合页未找到」页面，3s 后自动跳转到亚米首页

---

### 需求 10：分享与推广页（Share）

**验收标准：**
1. 展示推荐商品，提供品类 tab 切换功能
2. 点击商品分享按钮打开分享弹窗
3. 展示已创建的集合页列表，支持查看、编辑、分享
4. 支持创建新集合页

---

### 需求 11：创作者钱包（Wallet）

**验收标准：**
1. 展示收入交易列表（日期、商品、订单金额、佣金等）
2. 按月展示账单（待处理、处理中、已完成）
3. 账单里包含订单明细
4. 订单明细以 SKU 维度展示，并显示对应的购买数量、实付款、下单日期、佣金金额及佣金状态（已确认、待处理、已取消）
5. 送达 30 天后，则佣金状态=已完成；送达 30 天内，则佣金状态=待处理；送达 30 天内，取消订单或退换货，则佣金状态=已取消

---

### 需求 12：个人资料与设置

**验收标准：**
1. 个人信息（同步亚米账号信息）
2. 管理关联的社媒账号
3. 上传/更新税表
4. 更新品类偏好
5. Influencer 活跃用户展示 Power Creator 标签
6. 条款查看
7. 常见问题

---

### 需求 13：Power Creator 标签

**验收标准：**
1. Influencer 滚动 3 个月确认销售额 ≥ $300 时授予标签
2. 销售额低于 $300 时移除标签
3. Affiliate 无论业绩如何均不展示，但后台提供打开 Affiliate 活跃标签的功能
4. 在工作台问候区和个人资料页展示

---

### 需求 14：免费样品（Influencer 专属）

**验收标准：**
1. Influencer 活跃 + featureSamples 开启时展示可领样品
2. 每人每 30 天只能领取一个样品
3. 领取时以 $0 下单，选择已有地址或创建新地址即可下单成功。需支持地址联想、地址校验功能。
4. 已下单的样品生成亚米订单，同步到亚米订单系统
5. 可查看样品领取历史，并在历史清单中提供查看订单按钮，点击跳转到对应的亚米订单详情页
6. Affiliate 不展示样品功能
7. 不活跃或 featureSamples 关闭的 Influencer 无法领取

**错误处理：**
- 已售罄 → 提示「Sold Out」
- 已有未完成领取 → 提示「最近已领取一个样品，过短时间再来领取吧。」
- 不符合条件 → 说明资格要求（暂不符合领取条件）

---

### 需求 15：佣金计算

**验收标准：**
1. 默认佣金：参考文档 https://docs.google.com/document/d/1zmnIB2KZIDMwp_3eByfOlRTVYeZWGEKmWQ2MAOjOY28/edit?tab=t.jistg4y8t6e9
2. 支持按品类配置默认的佣金比例
3. **佣金计算基数**：按商品金额扣完积分和优惠券后的 order subtotal（不含税、不含运费）
4. 管理员可为个别创作者设置自定义佣金比例
5. **优先佣金（Priority Commission）**：支持为三级分类或单个 SKU 设置优先佣金比例，优先佣金需配置生效时间（起止日期），在生效期内覆盖默认品类佣金
6. **礼卡排除**：单纯购买礼卡（Gift Card）的订单不计算佣金
7. **与邀请好友互斥**：若订单使用了邀请好友优惠券，则整单不计算 influencer 佣金
8. **与跟买的处理**：若订单包含跟买商品，优先从订单中剔除跟买商品后对剩余商品正常计算佣金；若系统无法做到部分剔除，则整单与邀请好友同样做互斥处理，不计算 influencer 佣金

---

### 需求 16：打款处理

**验收标准：**
1. 累计佣金 ≥ $xx 才可打款
2. 未提交税表的创作者无法打款
3. 满足条件时纳入月度打款批次
4. 按创作者所在国家显示对应货币（USD / CAD）

---

### 需求 17：活跃状态计算

**验收标准：**
1. 使用滚动 3 个月窗口计算
2. 确认销售额 ≥ $300 → 活跃
3. 低于 $300 → 不活跃
4. 定期自动重新计算所有创作者

---

## 三、商品详情页

### 需求 18：商品详情页 Get Link

**用户故事：** 作为已登录的创作者，我希望在浏览亚米商品详情页时，能直接获取该商品的专属追踪链接，以便边逛边分享，无需切换到工作台。

**验收标准：**
1. 已登录且身份为创作者（Affiliate 或 Influencer）的用户，在亚米商品详情页可看到分享 icon 右侧的 "Get Link" 按钮
2. 点击 "Get Link" 后，生成包含该创作者唯一标识的追踪链接，并自动复制到剪贴板，同时展示复制成功的提示
3. 未登录用户或非创作者用户不展示 "Get Link" 按钮，不影响原有页面分享功能
4. 生成的链接与工作台分享弹窗生成的追踪链接归因逻辑一致

## 四、后台

### 需求 19：管理后台数据概览

**验收标准：**
1. KPI 卡片：创作者总数、待打款金额、活跃样品数、置顶商品数
2. 最新 Influencer 申请列表
3. 本月佣金 Top 创作者排名
4. 品类佣金分布图
5. 环比趋势指标（较上月增减）

---

### 需求 20：Influencer 申请审核

**验收标准：**
1. 申请列表含姓名、平台、粉丝数、品类、申请日期、状态
2. 状态筛选（待审核/已通过/已拒绝）
3. 统计卡片（待审核/已通过/已拒绝数量）
4. 通过申请 → 更新状态并授权访问
5. 拒绝申请 → 需选择预设原因或自定义原因
6. 支持批量通过和批量拒绝
7. 仅适用于 Influencer，Affiliate 无需审核

---

### 需求 21：创作者管理

**验收标准：**
1. 可搜索的创作者列表（姓名、类型、状态、活跃度、平台、总收入）
2. 类型筛选（Affiliate / Influencer / 全部）
3. 详情抽屉含 4 个 Tab：基础信息、功能权限、佣金比例、推广数据
4. 佣金比例 Tab：按品类调整佣金，展示修改历史（日期、旧值、新值、操作人）
5. 功能权限 Tab：样品功能开关、活跃状态展示（含 3 个月销售额进度条）
6. 批量管理：Affiliate 活跃度标签前台展示全局开关

---

### 需求 22：打款管理

**验收标准：**
1. 待打款列表含创作者、金额、账期、税表状态
2. 打款前验证税表已提交
3. 支持单笔和批量打款
4. Excel 导出（单人明细和批量报表）
5. 订单明细抽屉（展示该账期所有佣金订单）
6. **财务第三方对账系统**：对账提现所使用的财务第三方系统待后续确认，确认后需补充对接方案和字段映射

---

### 需求 23：样品管理

**验收标准：**
1. 样品列表含商品信息、领取时间、总数量/已领取数量
2. 添加样品时提供商品搜索（名称/SKU/品牌，品类/卖家类型/库存筛选）
3. 配置领取时间（起止日期）和最大领取数量
4. 编辑已有样品设置
5. 展示符合条件的创作者数量（活跃 + featureSamples 开启）
6. 支持移除样品
7. **双仓库存展示**：后台选择样品时，同时展示两个仓库（美国东仓/西仓）的库存数量，方便业务判断
8. **双仓兜底**：系统尽量保证两仓都有库存；若某一区域无库存，系统提示该区域用户不可领取

---

### 需求 24：推荐商品管理

**验收标准：**
1. 手动置顶商品列表，含有效期和操作人
2. 置顶时需设置有效期并记录操作人
3. 支持取消置顶
4. 默认推荐逻辑：按品类 Top 100 来取，每日 06:00 更新
5. 添加置顶商品：按搜索找到希望置顶的商品（名称/SKU/品牌，品类/卖家类型/库存筛选）

### 需求 25：优先佣金管理（Priority Commission）

**用户故事：** 作为运营管理员，我希望为特定三级分类或 SKU 配置优先佣金比例及生效时间，以便在活动期间激励创作者重点推广指定商品。

**验收标准：**
1. 支持以三级分类或单个 SKU 为维度设置优先佣金比例
2. 每条优先佣金规则需配置生效起止时间
3. 生效期内，优先佣金覆盖该品类/SKU 的默认佣金比例
4. 生效期结束后自动恢复默认佣金比例
5. 优先佣金列表展示：分类/SKU、佣金比例、生效时间、状态（未开始/进行中/已结束）、操作人
6. 支持编辑和删除未开始的优先佣金规则
7. 前台工作台首页同步展示进行中的优先佣金活动 Banner（见需求 6）

## 五、通用需求

### 需求 26：国际化

**验收标准：**
1. 前端：支持 6 种语言（EN、简中、繁中、韩、日、法语仅加拿大）
2. 美国显示 USD，加拿大显示 CAD
3. 语言设置应用于所有用户端页面

### 需求 27：追踪链接归因

**验收标准：**
1. 每个追踪链接嵌入创作者唯一标识
2. 集合页链接同样嵌入标识
3. 通过追踪链接购买时归因到对应创作者
4. 无效标识时正常处理购买但不归因，记录日志
5. **归因窗口（Attribution Window）**：首单按 7 天、复购按 3 天计算，均指从点击追踪链接到成交的时间
6. **跨 App 追踪**：支持第三方 App 内置浏览器打开 Yami 链接后跳转至亚米 H5 或 Yami App 的同设备追踪归因
7. **排重规则**：用户点击多个 influencer 链接时，按最近一次点击的 influencer 账号进行归因（Last Click）

### 需求 28：多国家支持

**验收标准：**
1. 一期仅上线美国站，美国业务跑通后再开放加拿大
2. 商品和样品按国别区分
3. 加拿大 influencer 邀请美国用户（或反之）正常结算佣金，但使用成交域名所属国家的 commission rate 进行计算

### 需求 29：响应式设计与无障碍

**验收标准：**
1. 三端响应式（桌面 ≥1024px、平板 768–1023px、移动端 <768px）
2. 移动端交互元素最小 44×44px
3. 文字对比度满足 WCAG AA（正文 ≥4.5:1，大字 ≥3:1）
4. 支持键盘导航（Tab、Enter、Space、Escape）
5. 语义化 HTML + ARIA 属性

### 需求 30：已登录用户访问 Landing Page 的路由规则

**验收标准：**
1. 未登录 / 未注册用户 → 正常展示 Landing Page
2. 审核中的用户 → 重定向到「审核中」状态页
3. 被拒绝且未点击「已知晓」的用户 → 重定向到「已拒绝」状态页
4. 被拒绝且已点击「已知晓」的用户 → 正常展示 Landing Page（可重新了解或以其他身份申请）
5. 已通过的 Affiliate / Influencer → 重定向到工作台首页
